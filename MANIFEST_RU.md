# Manifest Solink Workspace

Обновлено: 4 июля 2026

## Назначение

Solink Workspace - персональная инженерная AI-операционная система. Она должна стать центром управления проектами, задачами, агентами, документацией и локальными инструментами.

Система не должна зависеть от одной конкретной нейросети или одного приложения. Codex, ChatGPT, Hermes, Fugu, Gemini, Claude и будущие модели рассматриваются как подключаемые исполнители.

## Главный принцип

Человек занимается архитектурой, инженерными решениями, стратегией и новыми идеями.

Агенты забирают рутинную работу:

- поиск;
- код;
- документацию;
- автоматизацию;
- исследования;
- запуск задач;
- подготовку отчетов;
- проверку состояния проектов.

## Архитектурные слои

### 1. Command Center

Главный интерфейс. Базовая форма - HTML-приложение.

Показывает проекты, задачи, агентов, состояние системы, документацию, статус выполнения и журналы решений.

Интерфейс сам не запускает программы напрямую. Он отправляет команды локальному сервису.

### 2. Command Service

Локальный сервис, например на Python.

Принимает команды:

- `OpenProject`
- `RunCodex`
- `RunHermes`
- `BuildFirmware`
- `OpenSolidWorks`
- `CommitGit`
- `SearchResearch`
- `PrepareReport`

И уже сам запускает EXE, Python, Docker, Git, браузеры и другие инструменты.

### 3. Agent Layer

Подключаемые исполнители:

- Codex;
- ChatGPT;
- Hermes;
- Gemini;
- Fugu в будущем;
- Claude при необходимости;
- другие модели.

Агенты должны работать через понятные роли, а не через жесткую привязку к конкретной модели.

Базовые роли:

- `fast` - быстрые мелкие задачи;
- `smart` - сложное мышление и архитектура;
- `code` - программирование;
- `research` - поиск и анализ.

### 4. Workspace Files

Настоящая долговременная память хранится в файлах.

Базовая структура:

```text
SolinkWorkspace/
  README.md
  MANIFEST_RU.md
  PROJECTS_HUB_RU.md
  DECISIONS_LOG_RU.md
  ROADMAP_RU.md
  projects/
  knowledge/
  media/
  scripts/
  agents/
  templates/
  logs/
  archive/
```

## Правила контекста

1. Длинный чат не считается надежным хранилищем памяти.
2. Важные решения переносятся в `DECISIONS_LOG_RU.md`.
3. Текущий статус проектов хранится в `PROJECTS_HUB_RU.md`.
4. У каждого активного проекта должна быть карточка в `projects/`.
5. Перед началом работы новый агент читает `MANIFEST_RU.md`, `PROJECTS_HUB_RU.md` и карточку нужного проекта.

## Статусы проектов

- Норма
- Требует внимания
- Заблокирован
- Работает агент

## Momentum

Momentum показывает не процент готовности, а живость проекта:

- высокий - проект активно двигается;
- средний - проект жив, но без ежедневного темпа;
- низкий - проект лежит в ожидании;
- заморожен - сознательно отложен.

## Интерфейсная философия

Интерфейс не должен копировать Obsidian и не должен превращаться в хаотичную базу заметок.

Ориентир - современная инженерная панель управления: минимум лишнего, максимум полезной информации, быстрое понимание состояния всей системы.

## Смартфон

В будущем смартфон должен стать главным пультом управления.

Компьютер дома - исполнитель. Телефон - интерфейс.

Примеры команд:

- "Продолжай SolinkDesk"
- "Проверь новые статьи"
- "Запусти Codex"
- "Открой SolidWorks"
- "Сделай коммит"
- "Подготовь отчет"

## Модульность

Система проектируется так, чтобы через год можно было заменить модель, оркестратор, часть локального сервиса или перенести часть логики в облако без переписывания всего проекта.
