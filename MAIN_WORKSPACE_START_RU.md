## Важное исправление по 404

Была проблема: клики по проектам вели на неопубликованные `.md` файлы и давали 404.

Принято решение:

- левое меню не должно уводить на отдельные `.md`, если они не опубликованы;
- клик по проекту должен выделять карточку внутри текущего HTML-пульта;
- кнопки действий показывать только при наличии реальной опубликованной ссылки;
- если карточка еще не опубликована, показывать текст: `Карточка ещё не опубликована`.

Если 404 повторится, проверить:

- нет ли в `index.html` ссылок `projects/*.md`;
- нет ли в fallbackData ссылок на неопубликованные Markdown-файлы;
- загружены ли на GitHub свежие `index.html` и `projects.json` именно из `C:\Users\solin\Documents\Codex\workspace`.

## Принятое архитектурное решение

Solin Workspace - не просто страница, а будущий центр управления проектами.

Слои:

1. Markdown - долговременная память и решения.
2. JSON - компактные данные для интерфейса.
3. HTML - видимый пульт.
4. Позже Command Service - локальный безопасный сервис для изменения файлов и выполнения команд.

HTML сам по себе не должен запускать EXE и получать опасные права.
